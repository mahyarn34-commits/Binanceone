# ربات تلگرامی اسکن بازار کریپتو (Toobit) — استراتژی Price Action / Smart Money Concepts

این ربات به‌صورت خودکار:

1. جفت‌ارزهای پرحجم USDT رو در Toobit اسکن می‌کنه.
2. روند اصلی هر نماد رو در **4H** با EMA50/EMA200 + Market Structure (HH/HL/LH/LL, BOS, CHoCH) تعیین می‌کنه.
3. روند رو در **1H** با EMA20/50 + Structure + Volume تأیید می‌کنه. فقط وقتی 4H و 1H هم‌جهت باشن، وارد مرحله‌ی بعد می‌شه.
4. روی **M15** با یک سیستم امتیازدهی (نه شرط‌های سخت‌گیرانه‌ی all-or-nothing) سیگنال ورود می‌سازه:

    | مؤلفه | امتیاز |
    | --- | --- |
    | تأیید روند 4H | 20 |
    | تأیید روند 1H | 15 |
    | BOS در جهت معامله | 15 |
    | Liquidity Sweep | 20 |
    | داخل محدوده‌ی Supply/Demand | 10 |
    | هم‌جهت با VWAP | 5 |
    | تأیید حجم | 5 |
    | کندل تأییدی (Engulfing/Pin Bar) | 10 |
    | **جمع** | **100** |

   فقط وقتی امتیاز از آستانه‌ی قابل‌تنظیم `SIGNAL_SCORE_THRESHOLD` (پیش‌فرض ۷۰) عبور کنه، سیگنال BUY/SELL با Entry/Stop Loss/Take Profit صادر می‌شه.
5. نتیجه رو به تلگرام می‌فرسته — هم به‌صورت خودکار (هر ۱۵ دقیقه، قابل تنظیم) و هم با دستور `/scan`.

⚠️ **این ابزار صرفاً یک دستیار تحلیل خودکار است، نه توصیه‌ی مالی.** هیچ سیستمی ۱۰۰٪ دقیق نیست؛ مدیریت ریسک و تصمیم نهایی همیشه با خودتونه.

## ساختار پروژه

```

├── requirements.txt
├── Procfile
├── railway.json
```

## پیش‌نیازها

- یک اکانت تلگرام
- یک اکانت [Railway](https://railway.app) (رایگان برای شروع کافیه)
- **نیازی به API Key صرافی نیست** — دیتای قیمت از endpoint های عمومی Toobit گرفته می‌شه.

---

## مرحله ۱: ساخت ربات تلگرام با BotFather

1. توی تلگرام دنبال `@BotFather` بگرد و باهاش چت کن.
2. دستور `/newbot` رو بفرست.
3. یک اسم برای ربات انتخاب کن (مثلاً `Crypto Signal Bot`).
4. یک یوزرنیم انتخاب کن که باید به `bot` ختم بشه.
5. BotFather یک **توکن** بهت می‌ده — همون `TELEGRAM_BOT_TOKEN` هست.

---

## مرحله ۲: دیپلوی روی Railway

1. کل پوشه‌ی این پروژه رو در یک ریپازیتوری GitHub آپلود کن.
2. توی Railway: **New Project → Deploy from GitHub repo**.
3. برو به تب **Variables** و متغیرهای زیر رو اضافه کن (لیست کامل در `.env.example`):

    | Key | مقدار |
    | --- | --- |
    | `TELEGRAM_BOT_TOKEN` | توکنی که از BotFather گرفتی |
    | `TELEGRAM_CHAT_ID` | فعلاً خالی بذار |
    | `TOP_N_SYMBOLS` | `40` (اختیاری) |
    | `SCAN_INTERVAL_MINUTES` | `15` (اختیاری) |
    | `SIGNAL_SCORE_THRESHOLD` | `70` (اختیاری) |
    | `RISK_PER_TRADE_PERCENT` | `1.0` (اختیاری) |
    | `DAILY_MAX_LOSS_PERCENT` | `3.0` (اختیاری) |
    | `MAX_CONSECUTIVE_LOSSES` | `3` (اختیاری) |
    | `MIN_RISK_REWARD` | `2.0` (اختیاری) |
    | `ACCOUNT_BALANCE_USDT` | `1000` (اختیاری) |

4. Deploy کن.

---

## مرحله ۳: گرفتن Chat ID

1. توی تلگرام برو سراغ ربات خودت و بزن `/start`.
2. عددی که برمی‌گردونه رو کپی کن و توی Railway → Variables → `TELEGRAM_CHAT_ID` بذار.
3. سرویس رو Restart کن.

## دستورات ربات

| دستور | کاربرد |
| --- | --- |
| `/start` | فعال‌سازی + گرفتن chat_id |
| `/scan` | اجرای فوری اسکن بازار |
| `/help` | توضیح مختصر عملکرد ربات |
| `/risk` | وضعیت ریسک امروز (ضرر تجمعی، ضررهای متوالی) |
| `/result win` یا `/result loss` | ثبت نتیجه‌ی آخرین معامله برای پیگیری مدیریت ریسک |

## محدودیت‌ها و نکات مهم

- این ربات **معامله نمی‌کنه**، فقط تحلیل و سیگنال می‌ده. اجرای معامله و حجم پوزیشن نهایی با خودتونه.
- تشخیص Market Structure / Liquidity / Supply-Demand با روش‌های ساده‌شده (fractal, ATR-based) پیاده‌سازی شده و ممکنه با ابزارهای حرفه‌ای تفاوت جزئی داشته باشه.
- محدودیت‌های «ضرر روزانه» و «ضررهای متوالی» فقط با گزارش دستی نتیجه از طریق `/result` فعال می‌شن (چون ربات معامله رو خودش اجرا نمی‌کنه) و state بین ری‌استارت‌های ربات پایدار نیست.
- قبل از استفاده‌ی واقعی، حتماً چند روز نتایج رو مانیتور کن و با تحلیل دستی مقایسه کن.
